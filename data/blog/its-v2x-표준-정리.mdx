# ITS·V2X 표준 정리 — 데이터 사전부터 협력형 자율주행 메시지까지

ITS(지능형 교통 시스템) 데이터 등록소와 OID 레지스트리 관련 작업을 하면서, 기준으로 삼아야 할 국제·국내 표준을 한꺼번에 결제해둔 상태였습니다. 업무 중간중간 필요한 곳만 들춰보다가, 한 번쯤은 **"이 표준이 어떤 역할을 하고, 다른 표준과 어떻게 물려 있는지"** 를 지도처럼 정리해두는 편이 다음 작업에 훨씬 도움이 되겠다 싶어 이 글을 남깁니다.

다룰 표준은 총 14종입니다 (구입한 15종 중 `ITS데이터등록소구축.pdf`는 파일이 비어 있어 제외).

---

## 1. 왜 이걸 정리하는가

ITS 도메인의 표준은 **계층이 여러 겹**입니다. 하나의 V2X 서비스 한 개를 구현하려고 해도,

- 어떤 **메시지**를 주고받을지 (SAE J2735 / KS R 1600 시리즈)
- 메시지를 구성하는 **데이터 개념**을 어떻게 등록·관리할지 (ISO 14817-2)
- **센터 간** 또는 **센터–노변기기 간** 인터페이스는 어떤 규칙으로 오갈지 (KS X ISO 14827, 15784)
- 신호교차로 같은 **특정 애플리케이션 시나리오**에서는 어떤 조합을 쓸지 (ISO TS 19091)

가 모두 엮입니다. 그래서 하나의 표준만 읽으면 "이 용어는 어디서 왔지?" 가 자주 생깁니다. 각 표준의 **역할**과 **인용 관계**를 먼저 잡아두는 게 읽기 순서를 정하는 데 가장 유용했습니다.

---

## 2. 표준 체계 한눈에 보기

우선 14종을 **역할별로 4개 축**으로 나누면 다음과 같습니다.

| 축 | 표준 | 다루는 범위 |
|----|------|-------------|
| **데이터 사전 거버넌스** | ISO 14817-2:2015 | ITS 데이터 개념 등록소(CIDCR) 운영 절차 |
| **센터 간 인터페이스** | KS X ISO 14827-1 / 14827-2 | 메시지 정의 요구사항 / DATEX-ASN |
| **센터–노변기기 인터페이스** | KS X ISO 15784-1 / 15784-3 | 프로파일 원칙 / AP-DATEX 프로파일 |
| **V2X 메시지 사전 (글로벌 원형)** | SAE J2735 (2024-09) | V2X 전체 메시지·데이터 프레임·엘리먼트 사전 |
| **V2X 메시지 사전 (국가 표준)** | KS R 1600-1 ~ -7 | 협력형 자율주행용 V2X 메시지 명세 7부작 |
| **협력형 ITS 애플리케이션** | KS X ISO TS 19091:2019 | 신호교차로에서의 V2I / I2V 통신 이용 |

관계를 글로 풀면:

- **J2735가 전 세계 V2X 메시지의 원본 사전**이고, **KS R 1600 시리즈가 이를 국내 표준으로 체계화**한 것입니다.
- **14827/15784는 센터·노변기기 간 통신 인터페이스**를 규정하고, **14817-2는 이 모든 데이터 개념을 등록소에 어떻게 등재할지**를 규정합니다.
- **ISO TS 19091은 "신호교차로" 라는 특정 유스케이스**에서 위 메시지들을 어떻게 조합해서 써야 하는지를 풀어놓은 애플리케이션 표준입니다.

## 3. ISO 14817-2:2015 — ITS 데이터 개념 등록소 거버넌스

영문 원문으로 결제된 유일한 표준입니다. 내용이 가장 추상적이지만, **우리가 만들고 있는 "ITS 데이터 등록소" 의 지침서**에 가깝습니다.

**핵심 내용**

- Central ITS Data Concept Registry(CIDCR)에 데이터 개념을 **등록하는 프로세스**를 규정
- 단, **개념 모델**이지 **물리 구현**을 강제하지 않음 ("the meta model need not be physically implemented exactly as specified")
- 등록소 운영 주체의 역할을 세분화

| 역할 | 책임 |
|------|------|
| ITS executive committee | 최상위 의사결정 |
| Change control committee | 개념 변경 심의 |
| Registration Authority | 등록 권한 보유 |
| Registrar | 실무 등록 |
| Steward | 개념 품질 관리 |
| Submitter | 등록 요청자 |
| Read-only user | 조회 전용 |

**관리 메타 속성(Administrative meta-attributes)** 도 상세하게 정의되어 있습니다: create date, revision date, submitter, steward, owner, quality, status, follower, comment, harmonization candidate 등. 부속서 A~E에는 **버전 관리·품질 관리·상태 레벨·조화(harmonization)** 까지 규정되어 있어, OID Creation Wizard 같은 UI를 설계할 때 **어떤 필드를 반드시 받아야 하는지**의 출처가 됩니다.

> 즉 "ITS 데이터 등록소를 어떻게 만들 것인가" 라는 질문의 **최상위 참조문서**입니다.

## 4. 센터 간 인터페이스 — KS X ISO 14827-1 / 14827-2

### 4.1 KS X ISO 14827-1:2005 — 메시지 정의 요구사항

- **교통정보 센터들 사이에서 최종 응용 메시지(end-application message)를 문서화하는 형식**을 정의
- 형식은 DATEX-ASN, CORBA 등 다양한 응용 프로토콜에 적용 가능
- 각 시스템은 **양방향 데이터 교환**, **고수준 정의**, **예상 기능 정의**, **타이밍** 등의 요건을 충족해야 함

### 4.2 KS X ISO 14827-2:1997 (2022 확인) — DATEX-ASN

- 센터 간에 **관련 데이터를 end-application message로 교환**하는 구체적 프로토콜
- 서브스크립션 / 퍼블리케이션 형태로 메시지 정의
- end-application messages를 **데이터 패킷으로 패키지화**하는 방법과 교환 규칙·절차를 규정
- "DATEX-ASN 네트워크는 다수의 시스템으로 구성된다" — 즉 **Pub/Sub 기반 메시징 구조**로 이해하면 친숙합니다.

두 표준은 한 쌍으로 읽어야 합니다. **14827-1이 "메시지를 어떻게 문서화할지", 14827-2가 "문서화된 메시지를 어떻게 실제로 주고받을지"** 를 담당합니다.

## 5. 센터–노변기기 인터페이스 — KS X ISO 15784-1 / 15784-3

이 축은 **교통정보센터(TIC) ↔ 노변기기(RSE)** 사이의 통신 규정입니다. 중요한 점은 **차량탑재장치(OBU)와의 통신, 차량 내부 통신, 카메라 등 영상 전송은 제외**된다는 것.

### 5.1 KS X ISO 15784-1:2008 — 기본 원리와 프로파일 체계

- 정보·메시지 교환을 위한 **기본 원리와 문서화 체계, 응용 프로파일**을 규정
- 적용 대상:
  - 교통정보센터 ↔ 노변기기 모듈 간
  - 노변기기 모듈 ↔ 노변기기 모듈 간
- **프로파일(profile)** 이라는 개념이 핵심 — 여러 기본 표준을 조합해 특정 상황에 쓸 수 있는 **표준화된 부분집합**을 정의

### 5.2 KS X ISO 15784-3:2008 (2022 확인) — AP-DATEX

- **14827(DATEX-ASN)을 실제로 써먹기 위한 애플리케이션 프로파일**
- 적용 대상은 15784-1과 동일 (센터 ↔ 노변기기 / 노변기기 ↔ 노변기기)
- OSI 참조모델 기반의 시나리오와 요구사항을 담음

즉 15784 시리즈는 **"14827을 현장에 적용하려면 이런 프로파일을 써라"** 는 실행 가이드에 해당합니다.

## 6. KS R 1600 시리즈 — 협력형 자율주행 V2X 메시지 명세 7부작

국가 표준으로 가장 최근(2023 제정, 2025 대규모 개정/제정)에 정비된 핵심 시리즈입니다. **7부작이 하나의 세트**로 움직입니다.

| 부 | 제목 | 핵심 메시지 | 상태 |
|---|------|-------------|------|
| 1부 | 개념적 시나리오 | (V2X 메시지 목록 도출) | 2023 제정 |
| 2부 | 차량 기본 안전 메시지 | **BSM** | 2025 개정 |
| 3부 | 교통 신호와 지도 메시지 | **SPaT, TLS, MAP, NMEA 보정** | 2025 제정 |
| 4부 | 프로브 메시지 | **PVD, PDM** | 2025 제정 |
| 5부 | 여행자 정보와 노변 경고 메시지 | **TIM, RSA** | 2025 제정 |
| 6부 | 개인 안전 메시지 | **PSM** | 2025 제정 |
| 7부 | 안전 요청과 회피 경고 메시지 | **CSR, EVA, ICA** | 2025 제정 |

### 6.1 읽는 순서 — 1부부터 시작하라

1부(개념적 시나리오)는 **메시지 자체가 아니라 유스케이스**를 정의합니다. 여기서 "CAV 생성 데이터 활용 시나리오", "인프라 생성 데이터 활용 시나리오" 를 통해 **필요한 V2X 메시지 목록을 도출**합니다. 2~7부는 거기서 나온 메시지들을 **실제 스펙(프레임·데이터 엘리먼트)** 으로 쪼개 둔 형태입니다.

### 6.2 인코딩 — ASN.1 + UPER 기본

모든 메시지는 ASN.1로 정의되며, 별도 명시가 없으면 **UPER(Unaligned Packed Encoding Rules)** 로 인코딩됩니다. BER / DER / OER / XER / JER 도 언급되지만 기본값은 UPER입니다.

예를 들어 KS R 1600-2(BSM)의 데이터 사전 일부:

| 번호 | 항목 이름 | 설명 | M/O | 데이터 유형 |
|------|-----------|------|-----|-------------|
| 1 | `coreData` | 차량 안전 기본 정보(위치·방향·브레이크 등) | M | `DF_BSMcoreData` |
| 2 | `partII` | 차량 안전 관련 추가 정보 | O | (선택적 확장) |

**M/O (Mandatory/Optional)** 표기와 **DF\_ / DE\_ 접두사로 구분되는 데이터 프레임 / 데이터 엘리먼트** 구조는 J2735에서 그대로 이어받은 규약입니다.

### 6.3 각 부가 커버하는 응용

- **2부(BSM)** — 차량 자체 상태를 주변에 방송 (주변차량 인지의 기반)
- **3부(SPaT/MAP)** — 신호등 현시 + 도로 기하 (자율주행이 "이 신호등이 몇 초 뒤 바뀐다" 를 아는 근거)
- **4부(PVD/PDM)** — 차량이 수집한 도로·주행 환경 정보 수집 (프로브 데이터)
- **5부(TIM/RSA)** — 노변에서 차량으로 보내는 여행자 정보·경고
- **6부(PSM)** — 보행자·자전거·도로 작업자 등 취약 도로 사용자의 이동 정보
- **7부(CSR/EVA/ICA)** — 공통 안전 요청, 긴급차량 경고, 교차로 충돌 회피

## 7. SAE J2735 (2024-09) — V2X 메시지 사전의 원형

**KS R 1600 시리즈의 원본**에 해당하는 북미 표준입니다. 293쪽의 방대한 분량으로, V2X 통신에 쓰는 거의 모든 메시지·데이터 프레임·데이터 엘리먼트를 한 권에 집대성해 둔 **V2X 사전**입니다.

- 2024-09 리비전은 **2020 버전과의 역호환성을 복원**한 것이 특징 (2023 버전에서 깨졌던 호환성 복구)
- ASN.1 모듈이 표준과 함께 배포됨 — 최근 공개된 **확장 표준들의 ASN.1 모듈도 함께 포함**

**메시지 카탈로그(5장)** 에는 BSM, CSR, CCM, EVA, ICA, MSCM, MAP, NMEA, PSM, PSM2, PDC, PDM, PDR, PVD, RGA, RSM, RSA, RUCCM, RUCRM, RWM, RTCM, SDSM, SCPR, SCPS, SPAT, SRM, SSM, TAM, TUM, TLSM, TSPaT, TIM 등이 들어 있습니다. KS R 1600의 2~7부가 이 중 어떤 부분집합을 국가 표준으로 채택했는지 맞대보면 매핑이 분명해집니다.

**데이터 프레임(6장)** 예: `DF_AccelerationSet4Way`, `DF_BSMcoreData`, `DF_AntennaOffsetSet`, `DF_ApproachOrLane`, `DF_AdvisorySpeed` 등.

즉 **"국내 표준을 구현하다 모호한 필드를 만나면 J2735 원문을 역참조"** 하는 전략이 자연스럽습니다.

## 8. KS X ISO TS 19091:2019 — 협력형 ITS 신호교차로 유스케이스

232쪽 분량의 **응용 표준**입니다. 앞서 나온 메시지·데이터 개념들을 **"신호교차로에서의 V2I / I2V 통신"** 이라는 구체적 맥락에 어떻게 적용할지 규정합니다.

- 안전, 이동성, 환경 효율성 향상을 목적으로
- **노변 장비 ↔ 차량** 간 교환을 지원하는 **메시지·데이터 구조·데이터 요소**를 정의
- **시스템 엔지니어링 프로세스**를 통해 요구사항 ↔ 유스케이스 ↔ 메시지 정의의 **추적성(traceability)** 을 확보

구성은 "기본 규격 + 부속서" 형태이며, 안전·이동성·지속가능성 **유스케이스를 먼저 정의**한 뒤 그 유스케이스를 충족시키는 메시지로 역산(backward mapping)하는 패턴입니다. KS R 1600-1이 시나리오 → 메시지 도출 방식을 쓰는 것과 흐름이 같습니다.

## 9. 표준 간 인용 관계 지도

글로 풀었던 관계를 **의존성 그래프**로 정리합니다.

```
┌──────────────────────────────────────────────┐
│  ISO 14817-2  (데이터 개념 등록소 거버넌스)     │  ← 메타 계층
└──────────────────────────────────────────────┘
                    ▲
                    │ (등록 대상 데이터 개념의 출처)
                    │
┌──────────────────────────────────────────────┐
│  SAE J2735  (V2X 메시지 사전 — 글로벌 원형)     │
└──────────────────────────────────────────────┘
                    ▲
                    │ (부분 채택 / 국가 체계화)
                    │
┌──────────────────────────────────────────────┐
│  KS R 1600-1 ~ -7  (국내 V2X 메시지 명세 7부)  │
└──────────────────────────────────────────────┘
                    ▲
                    │ (신호교차로 유스케이스 적용)
                    │
┌──────────────────────────────────────────────┐
│  KS X ISO TS 19091  (C-ITS 신호교차로 V2I/I2V)│
└──────────────────────────────────────────────┘

── 별도 축 (센터·노변기기 인터페이스) ──
ISO 14827-1  (메시지 정의 요구사항)
    └─ ISO 14827-2  (DATEX-ASN 교환)
         └─ ISO 15784-1  (프로파일 체계)
              └─ ISO 15784-3  (AP-DATEX 프로파일)
```

## 10. 실무에 어떻게 연결되는가

정리하면서 실제 작업과 맞닿는 지점이 꽤 선명해졌습니다.

- **OID / 데이터 개념 등록 UI**를 만들 때 받아야 할 메타 속성의 **근거**는 ISO 14817-2 §8.
- **등록 상태 / 품질 / 버전 관리** 로직은 14817-2 부속서 B~D.
- **V2X 메시지 스키마**를 DB에 등재할 때 M/O 표기·DF/DE 구분·ASN.1 UPER 기본값은 KS R 1600-2 ~ -7 / J2735의 공통 규약.
- **센터 간 메시지 교환** 인터페이스를 설계할 일이 생기면 14827 + 15784 조합이 출발점.
- **특정 유스케이스(신호교차로 등)** 를 설명해야 할 때는 ISO TS 19091의 기능 모델·유스케이스 섹션이 훌륭한 교과서.

표준 문서는 처음에는 목차만 봐도 압도되지만, **각 문서의 "1. 적용범위"** 만 먼저 쭉 훑어도 이 지도가 어느 정도 그려집니다. 다음에 세부 구현에 들어가기 전, 해당 메시지/개념의 **원본이 어느 표준에 있는지**를 이 글에서 역참조할 수 있도록 남겨 둡니다.

---

## 정리

| 구분 | 한 줄 요약 |
|------|-----------|
| ISO 14817-2 | "ITS 데이터 개념을 등록소에 **어떻게 올리고 관리할지**" 의 메타 규약 |
| SAE J2735 | V2X **전체 메시지·DF·DE 사전**의 글로벌 원본 |
| KS R 1600-1 | V2X **시나리오**에서 필요한 메시지 **목록 도출** |
| KS R 1600-2 | **BSM** (차량 기본 안전) |
| KS R 1600-3 | **SPaT / MAP / TLS / NMEA 보정** |
| KS R 1600-4 | **PVD / PDM** (프로브) |
| KS R 1600-5 | **TIM / RSA** (여행자 정보 / 노변 경고) |
| KS R 1600-6 | **PSM** (개인 안전) |
| KS R 1600-7 | **CSR / EVA / ICA** (안전 요청 / 회피 경고) |
| KS X ISO 14827-1 | 센터 간 메시지 **정의 요구사항** |
| KS X ISO 14827-2 | 센터 간 **DATEX-ASN 교환 프로토콜** |
| KS X ISO 15784-1 | 센터–노변기기 **프로파일 원칙** |
| KS X ISO 15784-3 | 센터–노변기기 **AP-DATEX 프로파일** |
| KS X ISO TS 19091 | **신호교차로 V2I/I2V** 유스케이스 적용 |

표준은 한 번 읽고 끝낼 성질이 아니지만, 이렇게 **지도** 한 장을 그려두면 다음에 필요한 조항을 찾을 때 출발점이 생깁니다. 세부 스펙(데이터 사전 표, ASN.1 모듈)은 필요할 때 별도 글로 쪼개 정리할 예정입니다.

# BEM(Block, Element, Modifier) 방법론으로 유지보수성과 확장성을 향상시키는 방법

CSS의 BEM(Block, Element, Modifier) 방법론과 이를 사용하여 코드의 유지보수성과 확장성을 높이는 방법에 대해 설명해 주세요. 또한, 프로젝트에서 BEM을 적용한 경험이나 다른 CSS 네이밍 방법론을 사용한 경험이 있다면 말씀해주세요.

​

질문의도 입니다!

​

BEM(Block, Element, Modifier) 방법론을 통해 CSS 코드를 구조화하고, 유지보수성과 확장성을 높일 수 있는지 평가하려는 의도입니다. 이를 통해 CSS 네이밍 규칙을 제대로 적용하고, 팀에서 협업할 때 코드의 일관성을 유지할 수 있는 능력을 확인하고자 합니다. 또한, 실무에서 BEM이나 다른 CSS 네이밍 방법론을 적용한 경험을 통해 코드 관리 능력과 프로젝트에서의 실용적 적용 능력을 평가하는 의도도 있습니다.

---

### 모범 답안:

**BEM(Block, Element, Modifier) 방법론이란 무엇이며 이를 사용하여 CSS의 유지보수성과 확장성을 높이는 방법은 무엇인가요?**

BEM은 Block, Element, Modifier의 약자로, CSS 코드 구조를 더 체계적으로 만들기 위해 등장한 네이밍 방법론입니다. 이 방법론을 사용하면 코드의 가독성과 유지보수성을 크게 향상시킬 수 있습니다.

- **Block**: 독립적인 기능적 블록으로, 페이지에서 개별적으로 존재할 수 있는 단위를 의미합니다. 예를 들어, `header`, `menu`, `button`과 같은 큰 구조적 단위가 블록에 해당합니다.
- **Element**: Block의 구성 요소로, 해당 Block에 종속적인 역할을 합니다. 예를 들어, `header` 블록 안의 `logo`나 `menu` 블록 안의 `item`과 같은 것이 Element입니다. 네이밍은 `block__element` 형식으로 이루어집니다.
- **Modifier**: Block이나 Element의 상태나 스타일을 나타냅니다. 예를 들어, `button--primary`처럼 스타일이나 동작이 달라지는 경우 사용됩니다. 네이밍은 `block--modifier` 형식으로 이루어집니다.

**BEM을 사용했을 때의 장점**

1. **코드의 일관성**: BEM의 명확한 네이밍 규칙을 따르면, 각 요소의 역할과 관계가 분명해져 협업 시 코드의 일관성을 유지할 수 있습니다.
2. **유지보수성**: CSS 코드의 구조가 명확해지므로, 코드를 추가하거나 수정할 때 다른 부분에 영향을 주지 않도록 관리할 수 있습니다.
3. **확장성**: Block, Element, Modifier를 조합하여 새로운 스타일을 쉽게 추가할 수 있으므로, 프로젝트가 커질수록 확장성이 뛰어납니다.
4. **중복 방지**: 클래스명 충돌을 최소화할 수 있고, 전역적으로 적용되는 스타일과의 혼란을 줄일 수 있습니다.

**실무에서 BEM을 적용한 경험**
제가 참여한 프로젝트에서 BEM을 적용해 유지보수성과 확장성을 개선한 경험이 있습니다. 예를 들어, e-commerce 웹사이트의 제품 리스트를 관리하는 컴포넌트를 BEM 방식으로 작성하여, 각각의 리스트 항목과 그 내부 요소들(이미지, 가격, 설명)을 쉽게 관리할 수 있었습니다. Block은 `product-card`, Element는 `product-card__image`, `product-card__description`, Modifier는 `product-card--featured`와 같이 정의했습니다. 이 방식을 사용하여 개발자는 새로운 요구 사항에 맞게 코드를 쉽게 수정하고 확장할 수 있었으며, 전체 스타일 구조가 일관성을 유지하여 다른 팀원과의 협업에서도 큰 도움이 되었습니다.

**다른 네이밍 방법론과 비교**
또한, 다른 프로젝트에서는 BEM 대신 **OOCSS(Object Oriented CSS)**를 사용해 본 경험도 있습니다. OOCSS는 객체 지향적 원칙을 따르며, 구조와 스타일을 분리해 재사용성을 높이는 방법론입니다. 두 방법론 모두 CSS의 유지보수성과 확장성에 도움을 주지만, OOCSS는 스타일 재사용에 강점을 가지고 있고, BEM은 명확한 네이밍을 통해 코드의 구조적 일관성을 더 강화할 수 있습니다.

---

CSS 코드를 구조화하고 유지보수성을 높이는 것은 모든 개발자에게 중요한 과제입니다. CSS의 구조가 복잡해지면, 유지보수나 새로운 기능 추가 시 코드가 충돌하거나 불필요한 중복이 발생할 수 있습니다. 이를 해결하기 위한 방법 중 하나가 바로 **BEM(Block, Element, Modifier)** 방법론입니다.

### BEM이란?

BEM은 Block, Element, Modifier의 약자로, 각각의 역할을 명확히 구분하여 네이밍하고 구조화하는 CSS 방법론입니다. 이 네이밍 규칙을 따르면, 코드가 명확하고 일관되게 유지되어 협업과 유지보수 모두에 큰 장점을 제공할 수 있습니다.

- **Block**: 페이지의 독립적인 컴포넌트나 영역을 의미합니다. 이를 통해 블록 단위로 코드를 분리하고 관리할 수 있습니다.
- **Element**: Block 내에서 기능적 구성 요소입니다. Block에 종속되어 있어도, 각 Element는 독립적으로 스타일을 가질 수 있습니다.
- **Modifier**: Block이나 Element의 변형이나 상태를 정의합니다. 이를 통해 요소의 상태나 스타일에 변화를 줄 수 있습니다.

### BEM의 구조

예시로 버튼을 만들어보겠습니다:

```html
<button class="button button--primary">Primary Button</button>
<button class="button button--secondary">Secondary Button</button>
```

위 코드를 보면, `button`이 Block이고, `--primary`, `--secondary`는 Modifier로 버튼의 상태나 스타일을 조절하고 있음을 알 수 있습니다.

### BEM의 장점

1. **협업의 용이성**: 명확한 규칙을 따르므로 코드 가독성이 높아져 팀원 간의 협업이 쉬워집니다.
2. **유지보수성**: 새로운 요구 사항이 생겨도 기존의 구조를 깨지 않고 쉽게 스타일을 추가하거나 변경할 수 있습니다.
3. **확장성**: CSS 코드를 추가할 때도 기존 코드를 재활용할 수 있어 프로젝트가 확장될수록 코드가 깔끔하게 유지됩니다.
4. **중복 방지**: 전역 클래스나 스타일과의 충돌을 방지하며, 요소의 특성을 명확히 정의해 중복을 줄일 수 있습니다.

### 실무에서 BEM을 적용한 경험

저는 이전 프로젝트에서 BEM 방법론을 사용하여 대규모 e-commerce 플랫폼의 스타일링을 진행했습니다. 다양한 상품 카드, 리스트, 필터 등 복잡한 UI 요소를 구성하면서, BEM을 통해 각 요소의 역할을 명확히 하고 코드의 일관성을 유지했습니다. `product-card`, `product-card__image`, `product-card--featured`와 같은 네이밍을 적용하여, 코드의 유지보수성과 확장성을 크게 향상시켰습니다.

### 결론

BEM은 CSS 코드의 구조를 명확하게 하고, 프로젝트가 커질수록 유지보수성과 확장성을 높이는 데 강력한 도구입니다. 특히, 팀 단위로 협업하는 프로젝트에서는 일관된 네이밍 규칙을 통해 코드 충돌을 방지하고, 새로운 요구 사항을 쉽게 반영할 수 있습니다.

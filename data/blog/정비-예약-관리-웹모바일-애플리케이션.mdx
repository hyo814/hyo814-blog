## 개요

이 프로젝트는 고객의 정비 예약 요청을 확인하고 처리할 수 있는 웹/모바일 애플리케이션입니다.
사용자는 예약 목록을 조회하고, 개별 예약의 상세 정보를 확인한 후 예약을 확정하거나 불가 처리할 수 있습니다.
또한, 예약 가능 여부 처리는 로컬 저장소(Local Storage)를 통해 저장됩니다.

## 기능 개요

### 1. 예약 요청 리스트

- 해당 날짜의 정비 예약 건을 시간별로 확인할 수 있는 페이지
- 예약 건수를 시간 단위로 그룹화하여 표시
- 취소된 예약을 숨길 수 있는 필터 기능 (기본값: 활성화)
- 예약 목록을 조회할 수 있으며, 개별 예약을 선택하여 상세 정보를 확인 가능
- 고객 방문 횟수를 조회하여 신규 고객 여부를 표시
- 스크롤 위치를 기억하여 사용자 경험(UX) 개선 (세션 스토리지 활용)

### 2. 예약 요청 확인

- 선택한 예약의 상세 내용을 확인할 수 있는 페이지
- 고객 정보, 차량 정보, 결제 상세 정보 표시
- "예약 확정" 버튼: 예약 확정 팝업을 노출하여 확정 가능
- "예약 불가" 버튼: 예약 불가 팝업을 노출하여 불가 처리 가능
- 확정 또는 불가능 처리된 예약 정보는 로컬 저장소(Local Storage)에 저장됨

### 3. UI/UX 요소

- 날짜 네비게이션: 이전/다음 날짜 이동 기능 제공
- 좌우 슬라이드 애니메이션을 통한 화면 전환
- 스크롤 시 상단 네비게이션 고정 및 그림자 효과 추가
- 예약 카드 리스트는 고객 이름 가나다순 정렬
- 예약 가능 여부 처리 후 목록 화면 데이터 갱신
- 요청사항이 3줄 이상일 경우 말줄임(`...`) 처리하여 가독성 향상
- 네트워크 오류 발생 시 3초 후 자동 새로고침하여 데이터 불러오기

## 기술 스택

- **프레임워크**: Next.js
- **스타일링**: Tailwind CSS
- **언어**: TypeScript, JavaScript
- **상태 관리**: Recoil
- **데이터 저장소**: Local Storage, Session Storage

## 주요 기능 상세 설명

### 1. 예약 요청 리스트

- 특정 날짜의 예약 목록을 시간별로 그룹화하여 제공
- "취소된 예약 안보기" 스위치 제공
- 예약 요청 리스트에서 개별 예약을 클릭하면 상세 정보 페이지로 이동
- 취소된 예약은 상세 조회 불가
- 고객 방문 횟수를 조회하여 신규 고객 여부를 표시

### 2. 예약 요청 확인

- 예약의 상세 정보를 확인 가능
- "예약 확정" 또는 "예약 불가" 버튼을 통해 상태 변경 가능
- 예약 상태 변경 시 로컬 저장소(Local Storage)에 반영

### 3. 예약 확정 팝업

- "예약을 확정할까요?" 메시지
- "확정하기" 버튼 클릭 시 예약 확정 및 목록 갱신

### 4. 예약 불가능 팝업

- "작업이 어려우신가요?" 메시지
- "예약 불가능" 버튼 클릭 시 예약 불가 처리 및 목록 갱신

### 5. 상태 관리 (Recoil 활용)

- **selectedDateState**: 현재 선택된 날짜 상태를 관리
- **currentPageState**: 현재 페이지 정보를 글로벌 상태로 유지
- 예약 목록 및 상세 정보 페이지 간 상태 유지 및 공유

### 6. 오류 처리 및 예외 처리

- API 호출 실패 시 "예약 정보를 조회할 수 없습니다." 메시지 출력 후 일정 시간 후 자동 새로고침
- 예약 요청이 없는 경우 "확인된 예약 정보가 없습니다." 메시지 표시
- 데이터 로딩 중에는 `Loading` 컴포넌트 표시

### 7. 로컬 저장소 활용

- 예약 확정 또는 예약 불가 처리 시 `localStorage`를 활용하여 상태를 저장
- 저장된 예약 상태는 페이지 새로고침 후에도 유지됨

## 화면 구성

### 1. 예약 목록 페이지

- 날짜별 예약 리스트 조회
- 예약 시간과 건수 표시
- 예약 목록은 고객 이름 기준으로 정렬

### 2. 예약 상세 페이지

- 예약된 상품 정보 표시
- 고객 정보, 차량 정보, 결제 정보 표시
- 확정 및 불가능 처리 버튼 제공

# Hospital Dashboard 프로젝트 보고서

# [**https://github.com/hyo814/vital-check**](https://github.com/hyo814/vital-check)

### **프로젝트 개요**

- **프로젝트 기간**: 2024년 9월 20일 ~ 9월 30일
- **목적**: 병원에 입원한 환자의 최신 상태 데이터를 실시간으로 모니터링하고 관리할 수 있는 대시보드 개발
- **사용 기술**: TypeScript, React.js, Next.js, TailwindCSS, Recoil, React Query, json-server (mock server)
- **기여도**: 환자 데이터 필터링, 정렬, 무한 스크롤, 복사 기능 등 사용자 편의성을 높이는 핵심 기능 구현

### **프로젝트 주요 기능**

1. **실시간 환자 데이터 모니터링**:
    - 환자의 최신 상태 정보를 실시간으로 확인하며, 상태에 따라 데이터를 필터링하고 정렬하는 기능을 구현
    - 테이블 헤더를 클릭하여 관측 일자, 혈압 등 다양한 기준으로 데이터를 정렬
2. **무한 스크롤 구현**:
    - Intersection Observer API를 사용하여 페이지 단위로 환자 데이터를 불러오는 무한 스크롤 기능을 적용하여, 사용자 경험을 향상
    - 스크롤 시 마지막 데이터가 보일 때 자동으로 다음 데이터를 로딩
3. **복사 기능**:
    - 환자 ID를 빠르게 복사할 수 있도록, 복사 아이콘 클릭 시 `navigator.clipboard.writeText` API를 사용하여 복사 기능 제공
4. **유저 친화적 데이터 관리**:
    - 상태 필터링 및 정렬 기능으로 효율적인 환자 데이터 관리 가능
    - 체크박스를 통해 환자 상태별 필터링 제공

---

### **주요 구현 사항 및 기술 스택**

1. **상태 관리**:
    - **Recoil**: 전역 상태 관리를 통해 필터 및 정렬 상태를 효과적으로 관리
    - **React Query**: 서버와의 비동기 데이터 패칭과 캐싱을 관리하여, 환자 데이터의 효율적인 상태 유지
2. **UI 구성 및 스타일링**:
    - **TailwindCSS**: 사용자 친화적인 직관적 UI를 구축, 색상과 구성 요소 일관성을 유지
    - **컴포넌트 구조**: 테이블, 필터링, 정렬 등 주요 기능을 작은 컴포넌트로 분리하여 유지보수성 향상
3. **무한 스크롤 및 데이터 최적화**:
    - Intersection Observer API를 활용하여 무한 스크롤 기능을 구현하였고, useRef와 useCallback을 사용해 데이터 중복 패칭 방지 논리를 추가

---

### **프로젝트 파일 구조**

- **components**: 테이블과 필터링, 정렬 기능을 각각의 컴포넌트로 분리하여 관리
- **hooks**: `usePatientStatusCounts`, `usePatientsData` 등의 커스텀 훅을 사용해 React Query 기반의 데이터 관리 및 무한 스크롤 기능 구현
- **utils**: 데이터 포맷팅, 소수점 반올림, 날짜 형식 변환 등의 유틸리티 함수들로 코드 재사용성을 높임

---

### **프로젝트에서 배운 점**

1. **상태 관리의 중요성**:
    - Recoil과 React Query를 조화롭게 사용하여 전역 상태와 서버 데이터 관리를 분리하면서, 컴포넌트 간 상태 공유와 비동기 데이터 처리가 더욱 효율적으로 이루어짐을 경험
2. **무한 스크롤 성능 최적화**:
    - Intersection Observer API를 통한 무한 스크롤 구현을 통해 사용자 경험을 최적화하는 데 중요한 성능 개선 포인트를 배움
3. **UI/UX 디자인의 중요성**:
    - TailwindCSS를 활용하여 직관적이고 일관성 있는 디자인을 제공함으로써 CSS 프레임워크의 유용성을 실감

### **강조할 점**

- **데이터 처리 및 UI 최적화**: 대량의 데이터를 필터링 및 정렬하면서 사용자 경험을 고려한 UI 최적화를 구현
- **Recoil과 React Query의 조화로운 사용**: 전역 상태 관리와 서버 데이터 처리를 분리하여 두 라이브러리를 적절히 활용함
- **무한 스크롤 기능의 성능 최적화 경험**: Intersection Observer API와 최적화 로직을 적용한 무한 스크롤 경험

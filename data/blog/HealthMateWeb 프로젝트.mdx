# HealthMateWeb 프로젝트

### **프로젝트 개요**

- **프로젝트 기간**: 2024년 10월 1일 ~ 10월 8일
- **목적**: 사용자 맞춤형 건강 정보를 제공하는 온라인 플랫폼 개발
- **사용 기술**: React, TypeScript, Next.js, Material UI, CSS Modules
- **기여도**: 회원가입, 로그인, 맞춤형 건강 정보 제공, 걸음 수 확인 등의 주요 기능 구현

### **주요 기능**

1. **회원가입 및 로그인**:
    - **회원가입**: 이름, 이메일, 비밀번호 유효성 검사를 통해 회원가입 기능 구현. 비밀번호는 보안 규정을 준수하도록 영문 대소문자, 숫자, 특수문자를 포함해야 하며, 유효성 검사를 모두 통과해야 가입 가능.
    - **로그인**: 성공적으로 로그인하면 JWT 토큰을 발급받아 로컬 스토리지에 저장하여, 인증이 필요한 페이지에 접근 가능. 로그인하지 않으면 홈 화면 외 페이지 접근이 제한됨.
2. **맞춤형 건강 정보 제공**:
    - 로그인 후, 사용자가 걸음 수, 포인트 정보와 함께 맞춤형 건강 정보를 카드 형태로 확인 가능. 각 카드에는 영양제, 보충제, 피트니스 상품 등의 정보를 담고 있으며, 찜하기 버튼으로 원하는 정보를 저장할 수 있음.
3. **기타 기능**:
    - **로그아웃**: 사용자가 홈 화면에서 로그아웃하여 로그인 화면으로 돌아갈 수 있도록 구현.
    - **유효성 검사 및 데이터 검증**: React Hook Form과 커스텀 검증 함수로 이메일 형식, 비밀번호 복잡도 등의 검증을 통해 안정적인 사용자 경험 제공.

---

### **기술 스택 및 도구**

- **React**: 재사용 가능한 컴포넌트 구조로, 효율적인 사용자 인터페이스 개발
- **Next.js**: SSR과 SSG를 지원해 초기 로딩 시간을 단축, 빌드와 라우팅 간편화
- **TypeScript**: 타입 안정성을 통해 코드의 신뢰성을 높이고 오류를 예방
- **Material UI**: 디자인 일관성과 반응형 UI를 빠르게 구성할 수 있도록 다양한 UI 컴포넌트 제공
- **CSS Modules**: 각 컴포넌트별 스타일링을 모듈화하여 유지보수성과 스타일 충돌 방지

---

### **프로젝트 파일 구조**

- **components**: UI 요소를 구성하는 주요 컴포넌트, 카드, 네비게이션 바, 푸터 등으로 세분화하여 관리
- **pages**: 홈, 회원가입, 로그인, 오류 페이지 등의 각 페이지 컴포넌트 구성
- **utils**: 인증 로직과 데이터 유효성 검증을 위한 유틸리티 함수들이 포함
- **styles**: CSS 모듈을 통해 각 컴포넌트 스타일을 관리하여 일관된 디자인 유지

---

### **프로젝트에서 배운 점 및 강조할 점**

1. **React와 Next.js를 활용한 성능 최적화**:
    - 서버 사이드 렌더링(SSR)과 정적 사이트 생성(SSG)을 통해 페이지 로딩 속도를 크게 개선하여 사용자가 빠르게 맞춤형 건강 정보를 확인할 수 있도록 성능을 최적화.
2. **TypeScript를 통한 코드 안정성 강화**:
    - TypeScript를 사용하여 코드에서 발생할 수 있는 타입 오류를 사전에 방지하고, 유지보수성을 높일 수 있었음. 타입 오류와 불일치를 미리 발견해 수정함으로써 안정적인 코드 작성에 기여.
3. **Material UI를 활용한 UI 구성의 용이성**:
    - 일관된 디자인을 제공하면서 커스터마이징이 용이한 UI 컴포넌트를 사용해, 효율적으로 반응형 UI를 구성할 수 있었음. 사용자 친화적인 디자인을 빠르게 구현.
4. **CSS Modules로 스타일 충돌 방지**:
    - 각 컴포넌트에 독립적인 스타일을 적용하여, 스타일 간 충돌 없이 일관성 있는 디자인 유지 가능.
5. **유효성 검사와 데이터 검증 로직을 통한 사용자 경험 개선**:
    - 회원가입 및 로그인에서 React Hook Form과 커스텀 검증 함수를 사용해 사용자 입력값을 실시간으로 검증함으로써 안정적인 사용자 경험 제공.

---

### **기타 고려사항 및 성과**

- **프로젝트에 사용한 주요 라이브러리 선택 이유**:
    - **React**는 재사용 가능한 컴포넌트 구조와 빠른 렌더링 성능을 제공.
    - **Next.js**는 SSR과 SSG를 지원해 빠른 페이지 로딩과 SEO 최적화.
    - **Material UI**는 디자인 일관성을 유지하며 다양한 UI 컴포넌트를 쉽게 구현.
    - **TypeScript**는 타입 안정성을 통해 코드의 신뢰성을 높임.
- **서버 사이드 렌더링(SSR)을 활용한 성능 최적화**:
    - Next.js의 SSR을 통해 초기 페이지 로딩을 최적화하여, 사용자가 로그인 후 데이터를 빠르게 확인할 수 있도록 함.

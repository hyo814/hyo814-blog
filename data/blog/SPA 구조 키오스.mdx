# SPA 구조 키오스

프로젝트 개요

사용자가 제품을 선택하고, 할인 쿠폰을 적용한 뒤 결제할 수 있는 웹 애플리케이션을 개발하는 프로젝트입니다. 이 프로젝트는 React와 Recoil을 사용하여 SPA(Single Page Application) 구조로 구현되었으며, 결제 프로세스 전반에 걸쳐 사용자 경험을 향상시키는 데 중점을 두었습니다.

​

기술 스택 및 도구

- 프론트엔드: React, Recoil, Material-UI, TypeScript, CSS Modules
- 상태 관리: Recoil
- HTTP 요청: Axios
- 기타 도구: ESLint, Prettier (코드 스타일 관리), WebStorm (개발 툴)

​

주요 업무 및 프로젝트 세부 내용

1. 카테고리 정보 및 제품 목록 구현

- 카테고리별로 제품을 분류하여, 선택된 카테고리 내의 제품 목록을 화면에 동적으로 표시.

- 제품 목록에서 제품 클릭 시 결제 목록에 추가되며, 각 제품은 옵션 선택 기능을 포함.

​

2. 할인 쿠폰 적용 기능

- 사용자가 금액 할인 또는 비율 할인 쿠폰을 선택해 결제 금액에 적용할 수 있는 기능 구현.

- `/coupons` API를 통해 쿠폰 정보를 가져오고, 쿠폰 타입에 따라 적절하게 할인 적용.

​

3. 주문 수량 추가/삭제 기능

- 사용자는 선택한 제품의 수량을 조절할 수 있으며, 각 제품의 수량에 따라 결제 금액이 실시간으로 갱신.

​

4. 결제 내역 관리 및 총 결제 금액 표시

- 결제할 제품 목록에 추가된 아이템들의 총 금액을 계산하여 사용자에게 표시.

- 결제 버튼 클릭 시 결제 내역이 영수증 형식으로 모달 창에 표시.

​

상세 내용 및 배웠던 점

1. Recoil을 통한 상태 관리:

Recoil을 처음으로 도입하여 사용해 본 프로젝트였습니다. Recoil은 React의 Hooks와 긴밀하게 통합되어 있어, 복잡한 상태 관리를 간결하게 처리할 수 있었습니다. 특히, 원자(Atom)와 선택자(Selector)를 활용하여 각 컴포넌트의 상태를 독립적이면서도 효율적으로 관리하는 방법을 배웠습니다.

​

2. CSS Modules 사용:

CSS 클래스의 충돌을 방지하기 위해 CSS Modules를 사용하여 컴포넌트별로 스타일을 캡슐화했습니다. 이를 통해 프로젝트의 유지 보수성을 높였고, 스타일을 각 컴포넌트에 맞춰 재사용할 수 있었습니다.

​

3. Axios와 API 통신:

Axios를 사용해 `/products`, `/coupons` 등의 API로부터 데이터를 받아왔고, 이를 기반으로 제품 목록 및 쿠폰 적용 기능을 구현했습니다. 비동기 API 호출과 상태 업데이트에 대한 깊은 이해를 할 수 있었습니다.

​

4. SPA 구현을 통한 사용자 경험 개선:

SPA로 구현하여 페이지 새로고침 없이 사용자와 동적으로 상호작용할 수 있었습니다. 이를 통해 빠르고 부드러운 사용자 경험을 제공할 수 있었습니다.

​

기타 고려 사항

1. 다양한 디바이스 대응:

미디어 쿼리를 활용하여 모바일 환경에서도 잘 작동할 수 있도록 준비하였으며, 향후 다양한 기기에서의 테스트를 통해 더 나은 사용자 경험을 제공할 예정입니다.

​

2. 테스트 코드 작성 및 리팩토링:

테스트 코드 작성을 통해 기능의 안정성을 높일 계획이며, 리팩토링을 통해 코드의 유지 보수성을 지속적으로 개선할 예정입니다.

​

3. 결제 UI 개선 및 확장 가능성:

결제 UI는 사용자의 피드백에 따라 지속적으로 개선하고, 결제 수단의 다양화를 통해 사용자의 선택권을 늘릴 수 있는 가능성을 염두에 두고 있습니다.

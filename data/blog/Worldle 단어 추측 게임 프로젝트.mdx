# Worldle 단어 추측 게임 프로젝트

### **프로젝트 개요**

- **목적**: React와 Next.js, Recoil을 활용하여 5글자 영어 단어를 맞추는 단어 추측 게임을 구현. 제한된 시도 내에 단어를 맞추는 게임을 통해 학습과 재미를 제공
- **사용 기술**: React, Next.js, Recoil, CryptoJS, CSS Modules
- **기여도**: 암호화 처리, 단어 검증 API 연동, 게임 상태 관리, 로컬 스토리지 데이터 유지 등 주요 기능 구현

### **주요 기능**

1. **단어 맞추기 기능**:
    - 사용자는 5글자의 단어를 입력해 최대 6번의 시도로 정답을 맞춥니다.
    - 정답 단어는 CryptoJS를 사용해 암호화되어 URL에 포함되며, 게임 시작 시 복호화되어 보안이 강화된 환경에서 게임이 진행됩니다.
2. **단어 검증 API 연동**:
    - 사용자가 입력한 단어가 실제 단어인지 검증하기 위해 [dictionaryapi.dev](https://dictionaryapi.dev/) API와 통신
    - 유효하지 않은 단어 입력 시 "단어를 찾을 수 없습니다"라는 안내 메시지 표시
3. **게임 종료 후 통계 기능**:
    - 게임이 종료되면 플레이 시간, 승리 횟수, 시도 통계 등의 데이터를 화면에 표시
    - 데이터는 로컬 스토리지에 저장되어 재접속 시에도 상태가 유지됨
4. **로컬 스토리지를 통한 진행 상태 유지**:
    - 게임 도중 브라우저를 종료해도 현재 진행 상태가 유지되도록 로컬 스토리지에 데이터 저장
    - 저장되는 데이터에는 현재 추측된 단어, 진행 중인 행, 게임 보드 상태, 통계 데이터(플레이 시간, 승리 횟수, 승률, 시도 횟수 등)가 포함됨

---

### **기술 스택 및 도구**

- **React**: 컴포넌트 기반 구조로 유지보수성과 재사용성을 높이고, 상태 관리와 반응형 UI 구현에 용이
- **Next.js**: 서버 사이드 렌더링(SSR) 및 정적 사이트 생성(SSG)으로 성능을 최적화하고 빠른 페이지 로딩 구현
- **Recoil**: 컴포넌트 간 상태 공유와 성능 최적화를 통해 원활한 게임 진행을 가능하게 함
- **CryptoJS**: 단어 암호화 및 복호화를 통해 보안을 강화
- **CSS Modules**: 독립적인 스타일 적용으로 스타일 충돌 방지 및 유지보수성 향상

---

### **프로젝트에서 배운 점 및 강조할 점**

1. **React와 Next.js의 결합**:
    - React의 컴포넌트 기반 아키텍처와 Next.js의 SSR 및 SSG 기능을 결합하여 효율적인 개발이 가능했고, 성능을 극대화할 수 있었습니다.
2. **상태 관리 최적화**:
    - Recoil을 통해 상태 관리의 복잡성을 줄이고 성능을 최적화했습니다. 컴포넌트 간의 상태 공유와 업데이트가 원활하게 이루어져 복잡한 게임 로직을 구현하는 데 유리했습니다.
3. **보안 강화**:
    - CryptoJS를 활용하여 URL을 통한 정답 노출을 방지하고, 단어를 암호화하여 사용자 정보와 게임 데이터를 보호할 수 있었습니다.
4. **로컬 스토리지 활용**:
    - 게임 상태를 유지하기 위해 로컬 스토리지를 적극 활용해, 브라우저 종료 후에도 진행 상태가 유지되도록 데이터를 저장했습니다. 이를 통해 사용자 경험이 크게 향상되었습니다.
5. **반응형 UI 설계**:
    - 다양한 기기에서 일관된 사용자 경험을 제공하기 위해 반응형 디자인을 적용했으며, CSS Modules을 사용하여 컴포넌트별로 스타일을 독립적으로 적용해 관리 효율성을 높였습니다.

---

### **문제 해결 과정**

1. **로컬 스토리지를 활용한 데이터 유지 및 유실 방지**:
    - 로컬 스토리지를 통해 사용자가 게임을 중단하고 다시 접속해도 진행 상태가 유지되도록 데이터 저장 및 오류 처리를 추가했습니다. 데이터가 누락될 경우 초기 상태로 되돌리고 오류 메시지를 표시하여 게임을 재시작할 수 있도록 했습니다.
2. **Recoil을 사용한 상태 관리**:
    - Recoil은 React와의 호환성이 뛰어나고 컴포넌트 간 상태 공유가 용이하여, 복잡한 게임 로직에도 적합한 상태 관리 도구로 활용했습니다.
3. **API 요청 최적화 및 로컬 사전 데이터 활용**:
    - 단어 검증 API의 응답 속도를 개선하기 위해, 자주 사용되는 단어를 로컬에서 빠르게 검증하고 나머지 단어는 API 요청을 통해 검증하는 방식으로 처리했습니다.

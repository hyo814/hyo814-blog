# Next.js, Tailwind CSS, TypeScript를 활용한 차량 리스트 구현

## 개요
Next.js, TypeScript, Tailwind CSS를 사용하여 **차량 리스트 페이지**를 구현하는 방법을 소개합니다. 이 프로젝트는 모바일 환경을 우선적으로 고려하면서도 데스크톱에서도 최적화된 UI/UX를 제공합니다.

## 기술 스택
### 🚀 주요 기술
- **Next.js**: 서버 사이드 렌더링(SSR) 및 정적 사이트 생성(SSG) 지원
- **TypeScript**: 정적 타입 시스템을 활용한 안전한 개발
- **Tailwind CSS**: 효율적인 스타일링 및 반응형 UI 적용
- **React Query (또는 SWR)**: API 데이터 페칭 및 상태 관리

## 프로젝트 요구 사항
### 📌 UI 및 기본 기능
- 차량 리스트 페이지의 경로는 `/list`로 설정
- **모바일 우선 디자인** (최대 너비 `420px` 설정)
- 차량 목록과 다양한 필터 기능 제공

### 🚗 차량 정보 표시
각 차량의 정보에는 다음을 포함합니다:
- **이미지**
- **차량명**
- **가격** (100원 단위 반올림 및 할인율 적용 표시)
- **주행 거리** (1000km 이상일 경우 `만 km` 단위로 변환)
- **출시 연도**
- **지역** (지역명이 21자를 초과할 경우 `...`으로 축약)

### 🔎 필터 기능
필터 기능을 통해 사용자는 원하는 차량을 쉽게 찾을 수 있습니다.
- **차종 필터**: 경형/소형, 준중형, 중형/대형, 수입, SUV (다중 선택 가능)
- **지역 필터**: 서울/경기/인천, 제주, 부산/창원, 대구/경북, 대전, 광주 (다중 선택 가능)
- **가격 정렬 필터**: 낮은 가격순, 높은 가격순 (단일 선택)
- 필터 선택 시 dim 처리하여 리스트 스크롤 방지

### ⭐ 특수 필터
특수 필터는 항상 고정되어 있으며, `or 조건`과 `and 조건`을 조합하여 동작합니다.
- 예시:
  - `SUV` + `수입` 필터 → SUV 또는 수입 차량 표시
  - `SUV` + `제주` 필터 → 제주에 있는 SUV 차량만 표시
  - `SUV` + `제주` + `특가` → 제주 지역의 특가 SUV만 표시

### 🔄 API 요청 및 예외 처리
- 로딩 중에는 **스켈레톤 UI** 또는 로딩 인디케이터 표시
- API 요청 실패 시 **에러 메시지** 제공
- 페이지 리로드 후에도 **필터 상태 유지**
- 검색 결과가 없을 경우 사용자에게 안내 문구 제공

## 프로젝트 실행 방법
### 1️⃣ 패키지 설치
```sh
npm install
```
### 2️⃣ 개발 서버 실행
```sh
npm run dev
```
### 3️⃣ 빌드 및 배포
```sh
npm run build
npm start
```
